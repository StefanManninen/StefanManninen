## 👋 Stefan Manninen

Most systems don’t fail by accident.  
They fail by design.

I focus on understanding why.  

### 🧠 What I do
- Analyze systems — technical, structural and procedural
- Find hidden flaws in logic, implementation and assumptions
- Build solutions that are stable, predictable and maintainable  

### ⚙️ Approach
- Understand before building
- Question assumptions
- Fix root causes — not symptoms  
- Keep systems simple, predictable and observable
- Avoid unnecessary dependencies and hidden complexity

### 🤖 AI usage
- Used to analyze, question and validate
- Not to replace understanding
- No autonomous agents — systems must remain transparent and traceable 

### 🛠️ Tools
Languages and frameworks vary depending on the problem  
(JavaScript, Dart, Python, C/C++, etc.)  

### 📦 Projects

**🧩 Daily Tools**  
Modular system with a shared core.  
Closed source — security and control take priority over openness.  

**📧 Mail Infrastructure**  
Self-hosted mail system built for reliability and minimal intervention.  

**🏫 Skolgranskaren**  
Built from real cases.  
Focused on structure, traceability and accountability.  

### 💭 Mindset
> The world is full of hidden bugs —  
> in systems, in processes, and in assumptions.
